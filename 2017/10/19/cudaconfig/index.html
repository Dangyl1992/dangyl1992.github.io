CUDA环境配置

CUDA环境配置

CUDA全称是“Compute Unified Device Architecture”，中文名称即统一计算设备架构，它是NVIDIA公司提出了一种通用的并行计算平台和编程模型。使用CUDA，我们可以开发出同时在CPU和GPU上运行的通用计算程序，更加高效地利用现有硬件进行计算。为了简化并行计算学习，CUDA为程序员提供了一个类C语言的开发环境以及一些其它的如FORTRAN、DirectCOmpute、OpenACC的高级语言/编程接口来开发CUDA程序。

CUDA环境搭建

提示：推荐去NVIDIA官网查看官网文档，尽量不要Baidu。针对不同系统以及不同GPU的环境搭建略有不同，但大体的思路基本一致。

本文档主要采用离线方式在Ubuntu14.0.4 LTS系统上进行CUDA环境搭建。

准备工作

  1. 安装Ubuntu 14.0.4 LTS系统，安装完成之后尽量不要升级系统或者执行sudo apt-get upgrade命令;更新系统之后可能会产生一些不可预测的错误。

  2. 从官网下载对应的CUDA Toolkit，这里文档中下载的未cuda-repo-ubuntu1404-8-0-local-ga2_8.0.61-1_amd64.run。

    下载CUDA Toolkit时一定要对应自身系统环境！

  3. 从官网驱动下载界面选择对应版本的驱动程序进行下载。

    推荐*.run文件。本文中下载的为NVIDIA-Linux-x86_64-375.39.run

NVIDIA驱动安装

可选的步骤，不过建议老显卡还是手动安装驱动会比较好。

  1. 进入BIOS设置，从集成显卡进入系统。（默认情况下使用的是集成显卡）。
  2. 完全卸载NVIDIA驱动
    sudo apt-get remove --purge nvidia*

提示：有时候还可能需要卸载CUDA，需要在上面命令的基础上追加下面语句。
sudo apt-get autoremove

  1. 重启系统，进入系统时按ctrl+alt+F1组合键进入tty终端模式，登录系统，关闭lightdm

sudo service lightdm stop

提示：这里的命令仅适用于ubuntu，其他系统可自行查阅资料。

  1. 安装驱动
    安装内核头和开发包：
    sudo apt-get install linux-headers-$(uname -r)
    本文安装的为GT640M的驱动程序：

sudo sh NVIDIA-Linux-x86_64-375.39.run

可能会出现卡在登录页面无法进入系统的问题，这是需要卸载驱动，然后使用命令
sudo sh NVIDIA-Linux-x86_64-375.39.run -no-x-check -no-nouveau-check -no-opengl-files

重新安装驱动程序。

说明：
-no-x-check：安装驱动时关闭X服务
-no-nouveau-check：安装驱动时禁用nouveau
-no-opengl-files：只安装驱动文件，不安装OpenGL文件

运行命令之后，会看到相关的提示信息以及确认选项选项如Accept ;OK等，可能会报错The distribution-provided pre-install script failed!不必理会，一路Accept、ok、yes。

  1. 验证驱动是否安装成功
    cat /proc/driver/nvidia/version

  1. 启动图形环境
    输入命令
    sudo service lightdm start
    成功之后重启系统。
  2. 分辨率问题
    使用命令修改xorg.conf文件的参数sudo gedit /etc/X11/xorg.conf，修改结果如下所示：

这里添加了一条Modes "1024x768"；这里分辨率需要根据实际情况进行设置。然后重启系统即可。

CUDA Toolkit v8.0.61 安装

注意：8.0.61为随机选取的，需要根据自己的实际情况选取合适的版本进行学习和研究。

  1. 安装
    官网给出的安装步骤：
    $ sudo sh cuda-repo-<distro>_<version>_<architecture>.run

  2. 配置环境
    使用命令打开/etc/profile文件：
    sudo gedit /etc/profile
    在文件末尾追加：
    export LD_LIBRARY_PATH=/usr/local/cuda-8.0/lib64/:/usr/local/cuda/lib64:$LD_LIBRARY_PATH
    export PATH=/usr/local/cuda-8.0/bin:/usr/local/cuda/bin:$PATH

    reboot系统
    3.验证安装是否成功
    输入nvcc -V命令，出现如下结果则表示安装正确。

至此，安装和配置已全部完成。

CUDA C编程

CUDA C语言扩展

CUDA C语言对C语言的扩展主要包含以下几个方面：

  1. 函数限定符
函数限定符 说明 调用端
__devic__ 声明在设备上执行的函数 GPU
__globale__ 声明的函数成为kernel函数 CPU
__host__ 声明在主机执行的函数 CPU
  1. 变量限定符
变量限定符 说明
__devic__ 声明在设备上的变量, global memory 中，是全局变量
__constant__ 声明在常量存储中的变量，只读，只能在设备端使用
__host__ 声明在共享存储中的变量，仅供block内所有thread共享访存，退出kernel函数之后无效

  1. 内置数字变量
    一些特殊类型。如dim3类型，整型数组，用于指定线程(块)维度。

提示：详细请自行查阅资料

  • 内置变量

    变量 说明
    gridDim 指定grid维度
    blockDim 指定block维度
    blockIdx 指定grid内block索引号
    threadIdx 指定block内thread索引号
    warpsize 指定warp内thread索引号

  • kernel调用
    kernel函数调用就是调用声明为__global__函数，必须在主机端调用。调用时需要指定线程维度，<<Bs,Ts>>，其中Bs指定grid内block维度;Ts指定block内thread维度

  • 特殊函数和内建函数
    CUDA C中引入了大量专用的特殊函数和内置函数以实现特殊功能。
    提示：详细请自行查阅资料

    CUDA编程七部曲

    一个完整的CUDA代码需要包含7个关键步骤，尤其是多GPU的情况下;若单GPU的情况下，可以精简为5步。

    cudaSetDevice(0); //获取设备，只有一个GPU时或默认使用0号GPU时可以省略
    cudaMalloc((void **) &d_a, sizeof(float)*n); //分配显存空间
    cudaMemcpy(d_a, h_a, sizeof(float)*n, cudaMemcpyHostToDevice);//H2D数据传输

    gpu_kernel<<<blocks,threads>>>();//kernel函数

    cudaMemcpy(h_a, d_a, sizeof(float)*n, cudaMemcpyDeviceToHost);//D2H数据传输

    cudaFree(d_a); //free 显存空间

    cudaDeviceReset(); //重置设备。可以省略

    Demo


    /**
     * Vector addition: C = A + B.
     */

    #include <stdio.h>

    #include <cuda_runtime.h>

    __global__ void vectorAdd(const float *A, const float *B, float *C, int numElements)
    {
        int i = blockDim.x * blockIdx.x + threadIdx.x;

        if (i < numElements)
        {
            C[i] = A[i] + B[i];
        }
    }

    int main(void)
    {

        int numElements = 50000;
        size_t size = numElements * sizeof(float);
        printf("[Vector addition of %d elements]\n", numElements);

        // Allocate the host input vector A
        float *h_A = (float *)malloc(size);

        // Allocate the host input vector B
        float *h_B = (float *)malloc(size);

        // Allocate the host output vector C
        float *h_C = (float *)malloc(size);


        // Initialize the host input vectors
        for (int i = 0; i < numElements; ++i)
        {
            h_A[i] = rand()/(float)RAND_MAX;
            h_B[i] = rand()/(float)RAND_MAX;
        }

        // Allocate the device input vector A
        float *d_A = NULL;
        cudaMalloc((void **)&d_A, size);


        float *d_B = NULL;
        cudaMalloc((void **)&d_B, size);

        float *d_C = NULL;
        cudaMalloc((void **)&d_C, size);


        printf("Copy input data from the host memory to the CUDA device\n");
        cudaMemcpy(d_A, h_A, size, cudaMemcpyHostToDevice);
        cudaMemcpy(d_A, h_A, size, cudaMemcpyHostToDevice);

        cudaMemcpy(d_B, h_B, size, cudaMemcpyHostToDevice);

        int threadsPerBlock = 256;
        int blocksPerGrid =(numElements + threadsPerBlock - 1) / threadsPerBlock;
        vectorAdd<<<blocksPerGrid, threadsPerBlock>>>(d_A, d_B, d_C, numElements);


        // Copy the device result vector in device memory to the host result vector
        // in host memory.
        printf("Copy output data from the CUDA device to the host memory\n");
        cudaMemcpy(h_C, d_C, size, cudaMemcpyDeviceToHost);


        // Verify that the result vector is correct
        for (int i = 0; i < numElements; ++i)
        {
                printf("h_A[%d] = %f!-------------",i,h_A[i]);
                printf("h_B[%d] = %f!-------------",i,h_B[i]);
                printf("h_C[%d] = %f!\n",i,h_C[i]);
        }

        printf("CUDA kernel launch with %d blocks of %d threads\n", blocksPerGrid, threadsPerBlock);
        printf("Test PASSED\n");

        // Free device global memory
        cudaFree(d_A);


        cudaFree(d_B);

        cudaFree(d_C);


        // Free host memory
        free(h_A);
        free(h_B);
        free(h_C);

        printf("Done\n");
        return 0;
    }
